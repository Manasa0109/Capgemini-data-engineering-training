## Day 6 – Data Pipeline Implementation
 ### Overview

On Day 6, I focused on building and understanding data pipelines in PySpark, starting from a mini pipeline to a more advanced pipeline structure. The goal was to simulate a real-world data engineering workflow involving data ingestion, transformation, and processing.

### What I Learned

#### 1. Pipeline Concept
   
Understood how a data pipeline works end-to-end
Learned how to structure multiple transformations in a sequential flow
Realized importance of modular and reusable code in pipelines
#### 2. Data Ingestion
Reading data from files (CSV/structured sources)
Handling schema inference and column selection
#### 3. Data Transformation
Applied:
Filtering conditions
Column transformations
Case-When logic
Handling null values
Learned how to chain transformations effectively
#### 4. Mini Pipeline
Built a basic pipeline to:
Load data
Clean data
Apply transformations
Helped in understanding the step-by-step flow clearly
#### 5. Advanced Pipeline
Extended pipeline with:
Multiple transformations
Structured flow
Better readability and organization
Focused on writing scalable and cleaner code
#### 6. Phase 6 Implementation
Combined all concepts into a more complete pipeline
Simulated real-world data processing
Improved logical thinking in designing data workflows
### Workflow
Load data from source
Perform data cleaning
Apply transformations
Use conditional logic (case/when)
Handle null values
Generate final processed dataset
### Challenges Faced
Understanding how to structure pipeline steps properly
Managing multiple transformations without confusion
Ensing data flow correctness between steps
### Outcome
Gained clarity on how real-world data pipelines are built
Improved confidence in PySpark transformations
Learned how to move from a simple script → structured pipeline
