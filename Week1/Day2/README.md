## Day 2 – SQL Aggregations & Joins Practice

### Overview

On Day 2, I worked on SQL queries using Employee and Sales datasets.

I performed data insertion, aggregation, filtering using HAVING, and joins to analyze employee salaries and sales data.

This task helped me understand how SQL is used for real-world data analysis.

### What I Did

Inserted data into Sales table

Retrieved data using SELECT *

Performed aggregate functions:

SUM()
COUNT()
AVG()
MAX()
MIN()

Used GROUP BY to analyze department-wise data

Applied HAVING clause for filtering aggregated results

Performed INNER JOIN between Employee and Sales

### Used

ORDER BY
LIMIT
DISTINCT
YEAR() function

### What I Learned

Difference between WHERE vs HAVING

How GROUP BY works with aggregate functions

Writing queries for:

Total salary per department

Average salary

Employee count per department

Performing joins to combine multiple tables

Extracting insights like:

Highest sales

Department with highest revenue

Writing optimized SQL queries for analysis

### Challenges Faced

Confusion between WHERE and HAVING clause

Writing correct GROUP BY queries

Mistakes in JOIN conditions (missing ON condition sometimes)

Getting unexpected large outputs due to incorrect joins

Understanding how aggregation works after join

### How I Solved It

Practiced queries step-by-step.

Compared outputs to understand mistakes

Fixed join errors by properly using ON e.emp_id = s.emp_id

Re-ran queries and debugged logically

Practiced multiple variations of aggregation queries

### Outcome

Gained strong understanding of SQL aggregations and joins

Improved ability to solve real-world data analysis problems

Built confidence in writing interview-level SQL queries
